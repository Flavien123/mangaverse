Введение

В современном мире цифровые платформы для чтения манги становятся все более популярными. Существующие решения предоставляют обширные библиотеки контента, но часто не обладают удобным интерфейсом для персонализации и отслеживания прогресса чтения. 

Целью данного проекта является разработка полнофункционального веб-приложения для чтения манги с интуитивно понятным интерфейсом, системой персонализации и синхронизацией данных пользователя. Приложение должно обеспечивать комфортное чтение на различных устройствах и предоставлять инструменты для управления личной библиотекой.

1. Постановка задачи

1.1 Цель проекта

Главной целью проекта стало создание удобного веб-приложения для чтения манги. Сервис должен работать быстро, выглядеть современно и позволять пользователям не только читать, но и отслеживать свой прогресс. Важной задачей была реализация возможности работы на любых устройствах - от компьютера до смартфона.

1.2 Основные требования

При планировании проекта определились несколько ключевых функций. Во-первых, требовалась система регистрации и входа для пользователей, чтобы каждый мог иметь свой личный кабинет. Во-вторых, необходимо было дать возможность управлять своей библиотекой - отмечать мангу как прочитанную, добавлять в планы или откладывать на потом. Также важно было сохранять историю чтения, чтобы пользователь всегда мог вернуться к тому месту, где остановился. Приложение должно хорошо выглядеть и быстро работать на всех устройствах.

2. Анализ предметной области

2.1 Обзор существующих решений

Перед началом разработки изучались несколько популярных платформ для чтения манги. Большинство из них либо имеют устаревший интерфейс, либо работают только на мобильных устройствах. Многие сервисы не дают возможности гибко управлять своей библиотекой - например, нельзя отметить мангу как "отложенную" или "брошенную". Также часто отсутствует синхронизация между устройствами, что неудобно для современных пользователей.

[ВСТАВИТЬ ТАБЛИЦУ: Сравнение существующих решений с указанием их преимуществ и недостатков]

2.2 Выбор технологий

Для реализации проекта выбирались подходящие технологии. Для создания интерфейса использовался React - популярная библиотека, которая позволяет делать быстрые и отзывчивые приложения. Для стилизации выбран Tailwind CSS, с которым можно быстро создавать красивый дизайн без написания большого количества кода.

Серверная часть реализована на Python с использованием фреймворка FastAPI. Это современный инструмент, который работает очень быстро благодаря асинхронности. Для хранения данных использовалась PostgreSQL - надежная база данных, которая хорошо подходит для таких проектов.

Чтобы приложение было легко запускать на любом сервере, применялся Docker. Это позволяет упаковать все части приложения в контейнеры и запустить их одной командой.

[ВСТАВИТЬ ТАБЛИЦУ: Используемые технологии с кратким описанием назначения каждой]

3. Проектирование системы

3.1 Архитектура приложения

Приложение спроектировано по классической схеме клиент-сервер. Пользователь видит интерфейс, сделанный на React, который общается с сервером через HTTP запросы. Сервер обрабатывает эти запросы, работает с базой данных и отправляет ответы обратно. Такая архитектура позволяет легко разделить разработку на части и при необходимости масштабировать систему.

[ВСТАВИТЬ ДИАГРАММУ: Схема архитектуры приложения с указанием взаимодействия компонентов (Frontend, Backend, Database)]

3.2 Структура базы данных

В базе данных созданы несколько таблиц для хранения информации. Основная таблица - это пользователи, где хранятся логины, email и зашифрованные пароли. Отдельно создана таблица для статусов чтения - там записывается, какую мангу пользователь читает, какую планирует прочитать, а какую бросил. Есть таблица истории, где сохраняется последняя прочитанная глава и страница. И еще таблица оценок, где пользователи могут ставить баллы от 1 до 10 понравившейся манге.

3.3 API endpoints

Для взаимодействия клиента и сервера созданы несколько точек входа. Есть endpoints для регистрации и входа в систему, для получения информации о текущем пользователе. Отдельные endpoints позволяют получать и обновлять статусы чтения, историю и оценки. Все это работает через стандартные HTTP методы - GET для получения данных и POST для их обновления.

4. Реализация

4.1 Серверная часть

Серверная часть написана на Python. Созданы модели данных, которые описывают структуру таблиц в базе данных. Для безопасности реализована система аутентификации с использованием JWT токенов - это позволяет пользователям безопасно входить в систему и работать со своими данными. Пароли хранятся в зашифрованном виде, так что даже при получении доступа к базе данных их невозможно прочитать.

Весь код разделен на логические модули - отдельно аутентификация, отдельно работа с пользовательскими данными. Это делает код понятным и легким для поддержки.

4.2 Клиентская часть

Интерфейс приложения построен на React. Созданы несколько основных страниц - главная с новинками, каталог для поиска манги, страница с подробной информацией о выбранной манге, читалка для просмотра глав и личный кабинет пользователя.

Для управления состоянием пользователя использовался Context API - это позволяет всем компонентам приложения знать, вошел ли пользователь в систему или нет. Созданы отдельные модули для общения с сервером, чтобы весь код работы с API был в одном месте.

4.3 Дизайн и пользовательский опыт

При разработке дизайна использовались современные тренды. Выбрана темная тема, потому что она комфортнее для глаз при длительном чтении. В качестве акцентного цвета использовался светло-зеленый - он хорошо выделяется на темном фоне и создает приятный контраст.

Особое внимание уделено адаптивности. Приложение одинаково хорошо работает и на большом мониторе, и на телефоне. Для этого использовалась гибкая сетка и медиа-запросы. На мобильных устройствах некоторые элементы, например фильтры в каталоге, превращаются в выдвижное меню, чтобы не занимать место на экране.

Добавлены плавные анимации и переходы между состояниями - это делает интерфейс более живым и приятным в использовании. Когда данные загружаются, показываются специальные заглушки, чтобы пользователь понимал, что происходит.

5. Развертывание

5.1 Контейнеризация

Чтобы приложение можно было легко запустить на любом сервере, оно упаковано в Docker контейнеры. Созданы отдельные контейнеры для серверной части, для клиентской части и для базы данных. Каждый контейнер содержит все необходимое для работы своей части приложения.

[ВСТАВИТЬ ДИАГРАММУ: Схема Docker контейнеров и их взаимодействия]

5.2 Оркестрация сервисов

Для управления всеми контейнерами использовался Docker Compose. Это позволяет описать всю конфигурацию в одном файле и запустить все части приложения одной командой. В конфигурации указано, как контейнеры должны общаться друг с другом, какие порты открывать и какие переменные окружения использовать.

[ВСТАВИТЬ СКРИНШОТ КОДА: Пример docker-compose.yml]

5.3 Автоматизация развертывания

Написан простой скрипт, который автоматизирует процесс развертывания. Скрипт останавливает старые версии контейнеров, собирает новые образы и запускает их. Это очень удобно - не нужно помнить последовательность команд, достаточно запустить один файл.

В результате работы над проектом создано полноценное веб-приложение для чтения манги. Приложение имеет современный дизайн, удобный интерфейс и все необходимые функции для комфортного использования. Пользователи могут регистрироваться, управлять своей библиотекой, отслеживать прогресс чтения и ставить оценки.

Приложение получилось действительно адаптивным - оно одинаково хорошо работает на любых устройствах. Также достигнута хорошая производительность благодаря использованию современных технологий.

Особенно важным стало понимание того, как разные части приложения взаимодействуют друг с другом - как клиент общается с сервером, как сервер работает с базой данных, как правильно организовать код для удобства поддержки.

Использование Docker показало, насколько важна контейнеризация для современных приложений. Это значительно упрощает развертывание и делает приложение независимым от окружения.

Полученные знания и навыки будут полезны в дальнейшей работе над другими проектами. Понято, как важно правильно планировать архитектуру с самого начала, как тестировать приложение и как делать его удобным для пользователей.
